What is Sentiment Analysis?

Sentiment analysis is the process of using natural language processing (NLP) and machine learning to determine the emotional tone behind text data. It helps classify text as positive, negative, or neutral.

Generally, sentiment analysis is a tool that helps us understand the overall feeling or opinion behind a bunch of text, whether it's movie reviews, customer feedback, or social media posts. It's like having a mood detector for words! This can be really useful for businesses to see what people think about their products, for politicians to understand public opinion, or even just for you to get a quick summary of what people are saying about something online.

How it works
1. Fetching Social Media Data

APIs provide a bridge to gather essential data from social media platforms such as Instagram and Twitter. This involves generating API tokens and using Python libraries for HTTP requests.

Libraries Used:

  • Requests: Facilitates HTTP requests to fetch data from APIs.
  • TweePy: Provides easy access to the Twitter API.
Instagram Graph API:
  • Post Captions: Retrieve captions for sentiment analysis using the /me/media endpoint.
  • Comments: Analyze user comments via /media/{media-id}/comments.
  • Stories: Assess ephemeral content from /me/stories.
  • Hashtags: Identify trends using /ig_hashtag_search.
Twitter API:
  • Fetch Tweets: Collect tweets using the /tweets endpoint.
  • User Mentions: Track mentions for supportive or critical comments.
  • Hashtags: Analyze mental health-related hashtags like #selfcare, #depression, and #stress.

2. Data Preprocessing

Effective data processing involves cleaning and transforming social media text for analysis:

  • Remove unnecessary links, URLs, and special characters.
  • Normalize text by converting it to lowercase.
  • Tokenize text for efficient storage in databases.
  • Remove stopwords and handle missing values.

3. Sentiment Analysis

Sentiment analysis provides insights into the emotions conveyed in the text.

TextBlob Features:

  • Polarity and subjectivity sentiment evaluation.
  • Naive Bayes text classification.
  • POS tagging and noun phrase extraction.
  • Tokenization and spelling correction.
Algorithm:
  • Lexicon-based approach using predefined polarity and subjectivity scores.
  • Averaging scores for the entire text to derive sentiment insights.

4. Combining Insights

Integration of different data insights involves:

  • Normalization: Scale various metrics to a common range using Min-Max scaling.
  • Weight Assignment: Assign weights to different metrics (e.g., Sentiment Polarity at 40%).
  • Risk Score Calculation: Combine weighted metrics for a holistic score.
  • Threshold-Based Warnings:
    • Low Risk (0.0 - 0.4): No action needed.
    • Moderate Risk (0.4 - 0.7): Suggest positive mental health actions.
    • High Risk (0.7 - 1.0): Flag as a potential mental health concern.

5. Analyzing Behavior

Key behavioral analysis focuses on patterns and interactions:

  • Posting frequency
  • Engagement levels
  • Interaction trends
  • Time of activity
  • Hashtag usage
Steps for Analysis:
  • Data Extraction: Collect metadata and engagement data from APIs.
  • Data Aggregation: Organize data into daily, weekly, or monthly structures.
  • Normalization: Standardize metrics for fair comparison.
  • Trend Detection: Identify patterns in posting frequency and engagement levels.
  • Anomaly Detection: Spot sudden spikes or drops as potential concerns.
